Module 4

Wisconsin Pain Navigation Engine

Goal: turn clinical signals into localized Wisconsin support, not just a list of providers.

Back to Module 3 See all modules

Inputs used for routing

  • Pain phenotype from Modules 1 and 2
  • Language and insurance
  • ZIP code and rural status
  • Mobility constraints
  • Invalidation or distress markers

How it works

A geospatial recommender engine connects with 211 Wisconsin data and WIRE to produce ranked care pathways.

When Module 1 detects sleep issues and Module 2 shows high distress, the tool prioritizes Wisconsin Healthy Aging workshops or CBT-I providers that accept ForwardHealth.

Routing logic examples

  • Rural + high severity -> telehealth-first options
  • Medicaid -> ForwardHealth-accepting providers
  • High invalidation -> trauma-informed pain psychologist
  • Sleep cluster -> sleep clinic + CBT-I
  • Mobility barrier -> county transportation services
  • Language match -> culturally concordant providers

Output

Ranked care pathways plus non-pharmaceutical options such as PT, massage, acupuncture, and psychotherapy.

SDOH coverage

Built to address transportation, food access, and other social determinants that block care.
