The Chronic Pain Hub

A Multimodal Pain Communication & Navigation Platform

Empowering Patients Through Multimodal Communication

Our platform addresses the challenges of pain communication by providing multiple pathways for expression: words, images, sounds, and actionable navigation. Designed to bridge cultural, linguistic, and emotional barriers in chronic pain management.

Module 1: Semantic Translator/Pre-Visit Pain Communication Builder

Goal: To convert cultural metaphors and native-language narratives into medical ontologies.

How it works: Uses Cross-Lingual NLU (Natural Language Understanding). It doesn't just translate Hmong/Spanish; it performs Metaphor Mapping.

The AI Output: If a patient says "pain like a ghost in the marrow," the AI generates a clinician-facing note: "Patient describes deep, non-localized somatic distress (McGill Category: Affective/Exhausting)."

Outputs include 1-page "Visit Summary", Pain phenotype profile, Top concerns, Key questions for physician, Optional emotional/context flags.

Core Functions

  • Write or speak in preferred language
  • Guided prompts (dynamic, adaptive questioning)
  • Location selection (body map)
  • Animation-based pain quality selection
  • Context prompts (sleep, stress, food, flare pattern)

AI Layer

  • Cross-lingual semantic mapping (not literal translation)
  • Metaphor detection (e.g., "broken" in Hmong → subtype clarification prompts)
  • Clarifying follow-up prompts
  • Structured clinical summary generation

This module directly responds to challenges like "Hard to think of response", "Prepare ahead of time", "Numbers don't translate pain", "Doctors misinterpret symptoms", "Translation loses meaning".

Module 2: Visual Pain Expression System

Goal: To bypass linguistic barriers entirely using abstract animations.

How it works: Instead of asking "Describe your pain", patients choose from an Animation Taxonomy (e.g., Pulsing, Radiating, Crushing).

The AI Output: It correlates the visual choice with a Pain Phenotype.

Innovation: Detects Communication Discordance. If a patient's words are "calm" but they pick a "violent flashing" animation, the AI flags a "High Severity/Symptom Downplaying Risk" for the doctor.

Animation Categories (example taxonomy)

  • Pulsing / throbbing
  • Flashing / stabbing
  • Radiating outward
  • Crushing pressure
  • Static tightness
  • Electric shock
  • Diffuse fog-like ache

AI Role

  • Map animation selection → structured pain descriptor
  • Correlate visual + linguistic concordance
  • Detect mismatch (e.g., language mild, visual severe)
  • Cluster cross-cultural visual patterns (publishable)

Module 3: Low Burden Tracking/Bio-Acoustic Micro-Capture

Goal: To provide an objective pain "truth-check" through 5-second voice bursts.

How it works: Uses Acoustic Feature Extraction. The AI ignores what is said and analyzes how it is said (shimmer, jitter, vocal strain).

The AI Output: Generates a Vocal Strain Index (1.0–5.0).

Research Value: This is your strongest asset for a Federal R21 Grant, as it provides a new "digital biomarker" for pain intensity.

Core Design

  • One-tap flare button
  • Optional 5-second voice memo
  • Optional animation tap
  • No long forms

AI Layer

  • Extract key semantic features
  • Identify distress tone
  • Pattern clustering over time
  • Generate simple weekly pattern visualization

Module 4: Wisconsin Pain Navigation Engine

Goal: To turn clinical signals into localized Wisconsin support.

How it works: A Geospatial Recommender Engine integrated with the 211 Wisconsin API and WIRE (Wisconsin Information Referral Exchange).

The AI Logic: If Module 1 detects sleep issues and Module 2 shows high distress, the tool prioritizes Wisconsin Healthy Aging (WIHA) workshops or local CBT-I providers covered by ForwardHealth (Medicaid).

Innovation: It doesn't just list doctors; it solves for Social Determinants of Health (SDOH) like transport or food access.

Inputs

  • Pain phenotype (from Modules 1 & 2)
  • Language
  • Insurance
  • ZIP code
  • Rural/HPSA status
  • Mobility constraints
  • Invalidation/distress markers

Routing Logic Examples

  • Rural + high severity → telehealth-first options
  • Medicaid → ForwardHealth-accepting providers
  • High invalidation → trauma-informed pain psychologist
  • Sleep cluster → sleep clinic + CBT-I
  • Mobility barrier → county transportation services
  • Language match → culturally concordant providers

Other pain self-management tools: videos, non-pharmaceutical options for pain care—PT, massage, acupuncture, psychotherapy, etc.

Output

  • Ranked care pathway
